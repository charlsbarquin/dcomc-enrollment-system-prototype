## System Overview (Summary)

- **Backend:** Laravel; roles (admin, registrar, student, staff, unifast, dean); key models: User, FormResponse, Block, ScopeScheduleSlot, StudentCorRecord, CorScope, Assessment, EnrollmentForm, Program, Department, Subject, Room, Fee, FeeCategory, UnifastFeatureAccess, UnifastFeatureUserAccess, etc. `StudentCorRecord` has `cor_source`: `schedule_by_program` (Dean) or `create_schedule` (Registrar Irregular).
- **Flows:** Student enrollment (form → FormResponse → approve/reject/enroll); Registrar/Admin Student Status (filter students, edit record); **Schedule by Program** (Registrar and Dean) → scope slots → Dean deploys COR → StudentCorRecord (Dean COR Archive); **Irregularities → Create Schedule** (Registrar) → save/deploy → StudentCorRecord with `create_schedule` (Irregular COR Archive). Reports/Analytics with filters and CSV export; Block management; assessments (Staff/Unifast). **Fee Settings** (Registrar/Unifast): folder path = Program + Year Level; Raw fees list and Arrange-by-program/year table; fees stored with `program_id` and `academic_year_level_id` (see **docs/FEE_PROGRAM_YEAR_ISOLATION.md**).
- **Unifast role:** Managed under Registrar Settings → Unifast access (per-user and global toggles for Fees and Reports). Unifast has own dashboard, sidebar, and routes: Fee Settings (Raw/Arrange, Try button, add/delete/reorder, copy from raw), Reports, Assessments (eligibility, export). All Unifast actions use `unifast.*` routes; fees saved by Unifast use same `Fee` model and scope (program_id + academic_year_level_id) as Registrar.
- **COR by student type:** Regular, Transferee, Returnee see only **Dean COR Archive** (schedule_by_program). Irregular (and Shifter) see only **Registrar Irregular COR Archive** (create_schedule). Student dashboard “View COR” is blue for regular/transferee/returnee, yellow for irregular.
- **Fees on Student COR:** When the dean has deployed a schedule, COR records have `program_id` and `year_level` (AcademicYearLevel name). Student COR loads fees via `Fee::feesForScope(program_id, academic_year_level_id)` — the same scope as Fee Settings folder path. Unifast- or Registrar-saved fees for that program/year therefore appear accurately on the student COR (with fallback to block year level when COR record has no year_level).
- **Connections:** Dashboard cards link to Student Status with `process_status`; Student Status filters (student, level, program, block, shift, **process_status**); Form Responses use approve/reject (Registrar); Enroll/Reject/Needs correction/Delete application (AdminAccountController) use FormResponse id. All buttons, dropdowns, and fetches are wired to the correct routes and controllers.
